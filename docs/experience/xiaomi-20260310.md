## 编程
使用两个协程交替顺序打印奇偶数，上限设置为10或者20，方便运行观察结果。
<details>
<summary>点击查看答案</summary>
	```go title="main.go" showLineNumbers
	package main

	import (
		"fmt"
		"sync"
	)

	const UPPER = 10

	func main() {
		odd := make(chan struct{})
		even := make(chan struct{})
		var wg sync.WaitGroup

		wg.Add(2)
		// 打印奇数
		go func() {
			defer wg.Done()
			for i := 1; i <= UPPER; i += 2 {
				<-odd
				fmt.Println(i)
				even <- struct{}{}
			}
		}()

		// 打印偶数
		go func() {
			defer wg.Done()
			for i := 2; i <= UPPER; i += 2 {
				<-even
				fmt.Println(i)
				if i < UPPER { // 防止死锁
					odd <- struct{}{}
				}
			}
		}()

		odd <- struct{}{} // 启动
		wg.Wait()
	}
	```
</details>

## 算法
leetcode 17题 [电话号码的字母组合](https://leetcode.cn/problems/letter-combinations-of-a-phone-number/description/)
<details>
<summary>点击查看答案</summary>

思路一，笛卡尔积问题，使用 python 的生成器类型输出结果，避免内存占用。
```python title="python" showLineNumbers
from itertools import product

class Solution:
    def letterCombinations(self, digits: str) -> List[str]:
        phone = {
            "2": "abc",
            "3": "def",
            "4": "ghi",
            "5": "jkl",
            "6": "mno",
            "7": "pqrs",
            "8": "tuv",
            "9": "wxyz",
        }
        return ["".join(group) for group in product(*[phone.get(i) for i in digits])]
```
思路二，使用递归，求 n 位电话号码的字母组合，就是求后 n - 1 位电话号码的字母组合前面拼上第一位的结果。
```go title="go" showLineNumbers
var phone = map[string][]string{
	"2": []string{"a", "b", "c"},
	"3": []string{"d", "e", "f"},
	"4": []string{"g", "h", "i"},
	"5": []string{"j", "k", "l"},
	"6": []string{"m", "n", "o"},
	"7": []string{"p", "q", "r", "s"},
	"8": []string{"t", "u", "v"},
	"9": []string{"w", "x", "y", "z"},
}

func letterCombinations(digits string) []string {
	if len(digits) == 0 {
		return []string{}
	}
	if len(digits) == 1 {
		return phone[digits]
	}

	res := make([]string, 0)
	for _, i := range phone[string(digits[0])] {
		for _, group := range letterCombinations(digits[1:]) {
			res = append(res, i+group)
		}
	}
	return res
}
```

回表是什么
<details>
<summary>点击查看答案</summary>

在 InnoDB 存储引擎中，非主键索引的叶子节点包含的是主键的值。这意味着，当使用非主键索引进行查询时，数据库会先找到对应的主键值，然后再通过主键索引来定位和检索完整的行数据。这个过程被称为“回表”。
